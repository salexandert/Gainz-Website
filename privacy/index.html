Privacy model

Offline by default. Local files by design.

Gainz does not require a hosted account, exchange API sync, wallet sync, or transaction-history upload. It runs a local desktop/web interface on your computer and writes human-readable files.

No required transaction upload Reference-only tax evidence by default Local XLSX/CSV/Markdown outputs

Important limit

The local Gainz password protects the browser interface. It does not encrypt source CSVs, saves, exports, or audit packet files on disk.

What Gainz stores

Files stay where you put them.

Imported transactions

Imported CSV rows are stored in local Gainz save files so the app can reopen your review later.

Exports and packets

Generated XLSX, CSV, JSON, Markdown, and packet folders are written to a folder you choose.

Evidence references

Tax evidence scans default to reference-only records. The packet lists paths or labels without copying the files.

Explicit copies

Tax evidence files are copied into a packet only when you explicitly mark them for packet copy.

Network

What may touch the internet.

The reconciliation workflow itself does not require upload or cloud sync. Internet access may happen when you choose external actions such as opening the public website, GitHub releases, documentation, donation links, or update/download links.

Not uploaded by Gainz

Gainz does not upload transaction history, tax evidence, wallet exports, or audit packets to a Gainz server.

Synced folders

If you store files in OneDrive, iCloud, Dropbox, Google Drive, or another sync folder, that service may sync them.

Delete local data

You control the local files.

  1. Close Gainz.
  2. Back up anything you want to keep.
  3. Delete generated exports or audit packet folders you created.
  4. Delete local Gainz save, upload, log, and instance folders if you want to reset the app state.
  5. Remove files from any synced-folder trash or cloud recycle bin if they were stored in a sync service.

For the app-level walkthrough, open Privacy Mode inside Gainz. It shows the current local data folder paths for that installation.
