Website transparency

Privacy-preserving analytics for the public website.

Gainz needs basic public-site feedback: which pages are useful, whether downloads happen, and whether search engines can find the right pages. That does not require tracking app users or tax data.

Scope

Analytics described here apply to cryptogainz.store and public GitHub release counts, not the local Gainz desktop app.

Current approach

Aggregate website analytics only.

Website visits

Netlify Web Analytics can report aggregate page views and referrers from hosting logs without adding a client-side tracking script to this repo.

Release downloads

GitHub Release asset download counts are public aggregate numbers. Gainz snapshots those counts daily in the app repository.

Search indexing

Google Search Console and Bing Webmaster Tools are used to submit the sitemap and inspect indexing health.

No app telemetry

The local Gainz app does not upload transaction history, tax evidence, audit packets, or usage events to the website.

Provider rule

Keep analytics privacy-preserving.

If a client-side analytics provider is ever added, it should be a privacy-preserving option such as Plausible, GoatCounter, Cloudflare Web Analytics, or a similar cookieless aggregate analytics tool. Avoid ad pixels, remarketing tags, and broad behavioral tracking.

Search setup

Verification still needs account-owned tokens.

  1. Add https://cryptogainz.store/ in Google Search Console and Bing Webmaster Tools.
  2. Use DNS TXT verification when possible so verification is not tied to a file in the repo.
  3. Submit https://cryptogainz.store/sitemap.xml after verification.
  4. Use the search tools to watch indexing errors, not to track local app users.

Privacy line

Gainz can measure public website interest without collecting private reconciliation activity. Keep those worlds separate.
